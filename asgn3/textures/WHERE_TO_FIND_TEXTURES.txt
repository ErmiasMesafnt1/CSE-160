Where to get textures (for your WebGL world)
============================================

YOU ALREADY HAVE FILES IN THIS FOLDER
---------------------------------------
These are generated for you (64x64 PNG, power-of-two):
  grass_block.png   dirt_block.png   stone_block.png   planks_block.png
  cobble_paving.png sky.jpg
  (uvgrid.png kept as optional reference; slot 0 loads cobble_paving.png)

The game loads them from ../textures/ when you use a local server.
Regenerate the block PNGs anytime:
  cd asgn3
  python3 make_block_textures.py

To use planks instead of stone for slot 3: rename or swap files, or edit
World.js → upgradeWallAndSkyTexturesFromFiles() and change stone_block.png
to planks_block.png.


WEBSITES TO BROWSE (read each asset’s license before submitting work)
----------------------------------------------------------------------
1) Kenney (many free game packs; clear license on each pack page)
   https://kenney.nl/assets
   Try: search “voxel” or open “Voxel Pack” — download ZIP, pick PNGs.

2) OpenGameArt (filter by license: CC0 is easiest for class projects)
   https://opengameart.org/
   Use search: “16x16 tiles”, “32x32 texture”, “voxel”.

3) itch.io — Game assets, filter Free, read license on each page
   https://itch.io/game-assets/free

4) Poly Haven — mostly photo/PBR textures; fewer tiny tile sets
   https://polyhaven.com/textures


HOW TO USE A DOWNLOADED IMAGE
-----------------------------
- Prefer square size: 16, 32, 64, 128, 256, 512 (power of two).
- Put the PNG/JPG in this textures/ folder.
- Either replace an existing filename, OR add a new load line in
  asgn3/src/World.js in upgradeWallAndSkyTexturesFromFiles().

Do NOT use ripped Minecraft game files for published work; use packs that
say you may reuse them (CC0 / CC-BY with attribution, etc.).
